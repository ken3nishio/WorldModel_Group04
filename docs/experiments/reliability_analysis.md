# 実験計画の統計的信頼性分析 (Statistical Reliability Analysis)

## 1. 目的

現在計画している実験（各カテゴリ N=30〜50）において、提案手法（Step-Adaptive CFG / FAHC）の効果をどの程度の確度で検出できるか（統計的検出力）を定量的に評価する。
これにより、「実験結果が偶然ではない」と主張するための信頼性の根拠を提供する。

## 2. 前提条件

*   **検定手法**: 対応のあるt検定 (Paired t-test)
    *   同一プロンプト・同一シードで Baseline と Proposed を比較するため、個体差（プロンプトの難しさ）が相殺され、検出力が高くなる。
*   **有意水準 ($\alpha$)**: 0.05 (5%)
    *   誤って「効果がある」と判定してしまう確率。
*   **目標検出力 ($1-\beta$)**: 0.80 (80%)
    *   実際に効果がある場合に、正しく「効果がある」と判定できる確率。一般的に80%以上が望ましいとされる。

## 3. 検出力分析 (Power Analysis) の結果

### 3.1 検出可能な最小効果量 (Minimum Detectable Effect Size)

サンプルサイズ ($N$) ごとに、検出力0.80を達成するために必要な最小の効果量 (Cohen's $d_z$) を算出しました。

| サンプルサイズ (N) | 検出可能な効果量 ($d_z$) | 解釈 |
| :--- | :--- | :--- |
| **10** | 0.98 | **極大** (Huge): 劇的な変化がないと検出できない。予備実験レベル。 |
| **20** | 0.66 | **大** (Large): 明らかな改善が必要。 |
| **30 (計画値)** | **0.53** | **中** (Medium): 標準的な改善幅があれば検出可能。 |
| **50 (理想値)** | 0.40 | **中小** (Medium-Small): やや微妙な差も検出可能。 |
| **100** | 0.28 | **小** (Small): わずかな差でも有意になる。 |

### 3.2 Dynamic Degree (動き) に対する評価

VBenchの `Dynamic Degree` 指標（0.0〜1.0）において、標準的なばらつき（標準偏差 $\sigma$）を仮に **0.15** と推定します。
N=30 の場合、検出可能な効果量 $d_z = 0.53$ は、実数値の差分として以下に相当します：

$$ \Delta = d_z \times \sigma = 0.53 \times 0.15 \approx \mathbf{0.08} $$

つまり、**Dynamic Degree が平均 0.08 ポイント以上向上すれば、N=30 で統計的に有意な差として検出できます**。

*   **判定**: 消失タスク（Category B）では、「動かない(0.2)」→「消える(0.5)」のような劇的な変化（$\Delta \approx 0.3$）が予想されるため、効果量は $d_z > 2.0$ に達すると見込まれます。この場合、**N=30 での検出力はほぼ 100% (Power > 0.99)** となり、極めて高い信頼性があります。

### 3.3 Subject Consistency (一貫性) に対する評価

Category A（通常動作）において、「副作用がない（悪化していない）」ことを確認する場合です。
一貫性の許容範囲（非劣性マージン）をどう設定するかが鍵となりますが、微小な悪化（例: $\Delta = -0.02$）を「悪化」として検出するには、N=100以上のサンプルが必要です。

*   **判定**: N=30 では、**「劇的な悪化（$\Delta < -0.08$）」がないことは保証できます**が、「わずかな悪化」を見逃す（βエラー）可能性があります。
*   **対策**: 論文では「統計的に有意な悪化は観察されなかった（$p > 0.05$）」と記述しつつ、箱ひげ図で分布の重なりを可視化して定性的に補強するのが現実的かつ誠実な態度です。

## 4. 信頼性に関する結論

### **結論1: 消失タスク（検証の主眼）には十分すぎる信頼性がある**
提案手法の最大の目的である「消失・変形タスクでのダイナミクス向上」に関しては、予想される効果量が非常に大きいため、**N=30 のサンプルサイズで統計的に極めて堅牢な結論が得られます**。偶然の成功である確率はほぼゼロに近くなります。

### **結論2: 副作用の検証には一定の限界がある**
通常タスクでの副作用（一貫性の低下）については、N=30 では「明らかな崩壊」は検出できますが、「厳密な等価性」の証明には至りません。しかし、動画生成分野の慣例として、N=30〜50 は標準的あるいは多めの部類に入り、査読に耐えうる水準です。

## 5. 推奨アクション

1.  **実験実施**: Category A, B, C それぞれ **30件** ずつ（計90件）の生成を行う。
2.  **中間解析**: 最初の10件生成時点でt検定を行い、効果量の推定値を算出する。もし効果量が予想より小さければ（$d_z < 0.5$）、Nを50まで増やす判断を行う（Adaptive Design）。

---
**補足**: 
一般的に、CVPR等のトップカンファレンスにおける動画生成の評価では、N=50〜100程度のプロンプトセットが用いられますが、比較手法（Baseline vs Proposed）の差が明確（質的な変化）である場合、N=30程度でも十分に "Qualitatively and Quantitatively Superior" と主張可能です。
